## Professional Summary
A robust, browser-based Maintenance Management System (MMS) prototype designed to streamline facility maintenance workflows. This application facilitates seamless ticket submission, status tracking, and request management for both requesters and Facilities Management Services (FMS) teams, leveraging local storage for data persistence.

## 🛠️ Core Competencies Demonstrated
- **Frontend Development:** Architecting single-page application (SPA) behavior with native JavaScript.
- **Data Persistence:** Managing application state using `localStorage` for a zero-backend prototyping approach.
- **Workflow Automation:** Implementing role-based views (Requester/FMS) and automated ticket status updates.
- **UI/UX Design:** Creating intuitive, responsive interfaces with custom CSS and interactive modal components.
- **Dynamic Content Rendering:** Implementing search filtering, dynamic DOM manipulation, and visual data badges.

## 📂 Project Structure & Features
- `app.js`: Central logic controller handling authentication, data management, SPA routing, and dynamic UI rendering.
- `index.html` / `requester.html` / `fms.html`: Dedicated portals for user authentication, request submission, and management.
- **Features:**
  - Role-based Dashboard & Analytics.
  - Real-time ticket status tracking and FMS resolution workflow.
  - Dark/Light mode theme persistence.
  - Interactive calendar-based view of maintenance tasks.

## 🚀 Getting Started
1. Clone the repository.
2. Open `index.html` in any modern web browser.
3. Use the login interface to access either the Requester or FMS portal.
